# 编译器 - 新功能说明

## 功能一：编译真相架构

### 核心理念
笔记分区 = **结论区**（编译后）+ **记录区**（原始内容）

通过"编译"操作，将长篇笔记自动提炼为核心结论和关键洞察，帮助快速回顾。

### 数据结构
```typescript
interface CompiledSection {
  summary: string        // AI生成的核心结论
  insights: string[]     // 关键洞察列表
  lastCompiled: number   // 最后编译时间
  isManualEdit?: boolean // 是否手动编辑过
}
```

### 使用方法
1. **编译笔记**
   - 在笔记编辑器中，点击「编译笔记」按钮
   - 系统会自动分析笔记内容，提取核心结论
   - 编译结果会显示在编辑区上方的紫色区域内

2. **查看结论区**
   - 结论区包含：核心结论 + 关键洞察列表
   - 显示最后编译时间
   - 支持折叠/展开

3. **手动编辑**
   - 点击「编辑」按钮可修改 AI 生成的结论
   - 手动编辑后会标记为「已手动编辑」
   - 可随时点击「重新编译」刷新结论

4. **保存**
   - 编译结论会随笔记一起保存到 IndexedDB

### 模拟实现说明
当前为模拟实现（未接入真实 AI API）：
- 从笔记内容中提取前3句话作为"核心结论"
- 从内容中提取包含"关键"、"重点"、"核心"等关键词的句子作为"洞察"

## 功能二：梦境循环（夜间整理）

### 核心理念
在"夜间"自动整理笔记，检测问题并生成优化建议：
- 相似笔记（可能需要合并）
- 断裂链接（指向不存在的笔记）
- 整理建议（标签、归档等）

### 使用方法
1. **打开梦境循环**
   - 在侧边栏点击「🌙 梦境循环」按钮

2. **运行整理**
   - 点击「运行整理」按钮
   - 系统会分析所有笔记，生成整理报告

3. **查看报告**
   - **统计概览**：显示发现的问题数量
   - **整理建议**：可点击跳转到相关笔记
   - **相似笔记**：显示相似度高的笔记对
   - **断裂链接**：显示指向不存在笔记的链接

4. **执行建议**
   - 点击建议卡片可跳转到相关笔记
   - 根据建议进行合并、修正链接等操作

### 梦境循环检测项

| 检测项 | 说明 |
|--------|------|
| 相似笔记 | 标题或内容关键词重叠度 > 60% 的笔记对 |
| 断裂链接 | `[[笔记名]]` 引用不存在的笔记 |
| 无标签笔记 | 没有添加标签的长笔记 |
| 长期未更新 | 30天以上未修改的笔记 |

### 显示上次整理时间
侧边栏梦境循环入口会显示上次运行时间。

## 技术实现

### 文件结构
```
src/
├── components/
│   ├── NoteCompiledSection.tsx  # 结论区组件
│   ├── DreamCycleReport.tsx      # 整理报告组件
│   └── Sidebar.tsx               # 修改：添加梦境循环入口
├── lib/
│   ├── compiler.ts               # 编译模块
│   └── dreamCycle.ts             # 梦境循环核心逻辑
└── types/
    └── index.ts                  # 修改：添加新类型定义
```

### 预留 API 接入接口
- `lib/compiler.ts` 中的 `compileNote()` 函数预留了 AI API 接入接口
- `lib/dreamCycle.ts` 中的 `detectSimilarNotes()` 和 `detectBrokenLinks()` 函数可替换为更智能的 AI 分析

## 数据持久化
- 所有数据存储在 IndexedDB 中
- 笔记（包括编译结论）持久化保存
- 上次梦境循环运行时间保存在 localStorage
