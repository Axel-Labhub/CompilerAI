## 1. 项目概述

### 1.1 产品定位

**编译器**是一个专注于职场知识管理的本地笔记应用，核心理念是"你是你唯一的算法"。

区别于传统知识管理工具，编译器不做"第二大脑"，不卷大模型，不卖方法论。它只是一个安静的工具，帮用户把脑子里的碎片整理清楚，关键时刻能找到、想用时能调出来。

**核心差异**：
- 极简主义：下载即用，无需配置
- 本地优先：数据完全可控，不上云、不泄露
- 场景驱动：专为"离职交接"和"周报写作"设计

### 1.2 核心功能矩阵

| 功能 | 类型 | 描述 | 优先级 |
|------|------|------|--------|
| 本地Markdown编辑器 | 基础 | 实时预览、全屏编辑、自动保存 | P0 |
| 情绪清洗 | AI | 将情绪化文字转专业表述 | P0 |
| 一键周报 | AI | 从笔记自动生成周报 | P0 |
| 双向笔记 | 协作 | 笔记自动关联，`[[标题]]`语法 | P0 |
| 版本历史 | 协作 | 最多20版本，支持回溯对比 | P0 |
| 编译真相架构 | AI | 结论区+记录区分区管理 | P1 |
| 梦境循环 | AI | 夜间自动整理笔记 | P1 |
| 智能搜索 | 基础 | 全文索引+筛选，<0.5s响应 | P1 |
| 笔记模板 | 基础 | 会议纪要/日报/周报/复盘 | P2 |
| 深色主题 | 基础 | 亮色/暗色/跟随系统 | P2 |
| 导出功能 | 基础 | Markdown/HTML/PDF/剪贴板 | P2 |

### 1.3 目标用户画像

**核心用户**：
- **离职交接期的打工人** - 痛点：交接文档散乱、周报写不出、情绪输出多
- **刚入职的程序员/PM** - 痛点：需要快速了解项目、记不住技术细节
- **终身学习者** - 痛点：笔记越记越多，回头找不着

**用户特征**：
- 不愿意学习复杂的笔记工具（如Roam Research）
- 想要一个"安静"不打扰的写作环境
- 对数据隐私敏感，不信任云端存储

### 1.4 竞品对比分析

| 维度 | 编译器 | Obsidian | Notion | Typora |

### 1.5 编译器差异化优势

1. **情绪清洗** - 唯一内置情绪词映射的工具，写周报不用憋词
2. **一键周报** - 选笔记→出周报，0门槛
3. **编译真相架构** - 结论区+记录区分区，沉淀知识而非堆砌笔记
4. **极简启动** - 下载即用，无需学习复杂语法或配置

## 4. 数据模型

### 4.1 核心类型定义

```typescript
// 笔记 (Note)
interface Note {
  id: string                    // 唯一标识，格式: note_{timestamp}_{random}
  title: string                 // 笔记标题
  content: string               // Markdown 内容
  tags: string[]                // 标签列表
  createdAt: number             // 创建时间戳
  updatedAt: number             // 更新时间戳
  isPinned: boolean             // 是否置顶
  isFavorite: boolean           // 是否收藏
  compiledSection?: CompiledSection | null  // 编译分区
}

// 编译分区 (CompiledSection) - 编译真相架构核心
interface CompiledSection {
  summary: string       // AI 生成的核心结论
  insights: string[]   // 关键洞察列表
  lastCompiled: number // 最后编译时间
  isManualEdit?: boolean // 是否手动编辑
}

// 版本历史 (NoteVersion)
interface NoteVersion {
  id: string            // 唯一标识
  noteId: string        // 对应笔记 ID
  content: string       // 快照内容
  title: string         // 快照时的标题
  createdAt: number     // 创建时间
}

// 笔记链接 (NoteLink)
interface NoteLink {
  id: string            // 唯一标识
  sourceId: string      // 源笔记 ID
  targetId: string      // 目标笔记 ID
  targetTitle: string   // 目标笔记标题
  createdAt: number     // 创建时间
}

// 周报 (WeeklyReport)
interface WeeklyReport {
  title: string
  content: string
  summary?: string
  highlights?: string[]
  achievements?: string[]
  problems?: string[]
  nextWeek?: string[]
  generatedAt: number
}

// 笔记模板 (NoteTemplate)
interface NoteTemplate {
  id: string
  name: string
  description: string
  content: string
  icon: string
}
```

```typescript
// 数据库配置
const DB_NAME = 'compiler-db'
const DB_VERSION = 2

// 数据库表结构
interface CompilerDB extends DBSchema {
  // 笔记表
  notes: {
    key: string           // id
    value: Note
    indexes: {
      'by-created': number    // 创建时间索引

      'by-title': string      // 标题索引
      'by-favorite': number  // 收藏状态索引
    }
  }
  
  // 版本历史表
  versions: {
    key: string           // id
    value: NoteVersion
    indexes: {
      'by-note': string   // 关联笔记索引
      'by-created': number  // 创建时间索引
    }
  }
  
  // 链接表
  links: {
    key: string           // id
    value: NoteLink
    indexes: {
      'by-source': string   // 源笔记索引
      'by-target': string   // 目标笔记索引
    }
  }
}
```

## 13. 性能优化

### 13.1 IndexedDB 优化

- **批量操作**：使用事务批量处理
- **索引优化**：合理创建索引，避免全表扫描
- **增量存储**：版本历史使用增量存储节省空间

### 13.2 搜索优化

- **防抖**：搜索输入防抖 300ms
- **缓存**：缓存搜索结果
- **分页**：大结果集分页加载

### 13.3 渲染优化

- **React.memo**：避免不必要的重渲染
- **虚拟列表**：笔记列表使用虚拟滚动
- **懒加载**：按需加载组件

### 13.4 性能指标

| 指标 | 目标值 | 实际值 |
|------|--------|--------|
| 首次加载 | < 1s | ~800ms |
| 搜索响应 | < 0.5s | ~300ms |
| 笔记保存 | < 100ms | ~50ms |
| 内存占用 | < 100MB | ~50MB |

### 14.2 激活码方案

**技术实现**：
1. 用户通过 Gumroad/Lemon Squeezy 购买
2. 支付后自动发送激活码到邮箱
3. 用户输入激活码，首次联网验证
4. 验证通过，本地存储激活状态
5. 之后离线使用，无需持续联网

**防破解策略**：
- 激活时联网验证（一次性）
- 季度打卡机制（可选）
- 激活码+设备指纹绑定
- 一码最多3台设备

**平台选择**：
| 平台 | 抽成 | 优点 |
|------|------|------|
| Gumroad | 5% + 支付费 | 国际化、成熟、自动化程度高 |
| Lemon Squeezy | 5% + 支付费 | 专为软件设计、退款政策好 |
| 自己开发 | 0%（但需资质） | 完全可控，但需ICP许可证 |

**推荐**：Gumroad 或 Lemon Squeezy，0门槛上线，快速验证市场。

### 14.3 轻账号系统设计

**设计原则**：
- 用户下载即用，基础功能完全免费
- 只有在需要 AI 功能时，才弹窗引导注册
- 1分钟完成注册，不设门槛

**注册流程**：
```
用户点击AI功能 → 弹窗引导注册 → 手机号/邮箱 + 验证码 → 付费激活
```

**技术实现**：

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 短信服务 | 阿里云/腾讯云 | 国内首选，支持手机号验证码 |
| 邮箱服务 | SendGrid/Mailgun | 海外用户备用 |
| 后端服务 | Edge Functions | 轻量、无状态、全球部署 |
| 数据库 | Redis + PostgreSQL | 验证码缓存 + 用户存储 |

**短信验证码服务商对比**：

| 平台 | 单条价格 | 优点 | 缺点 |
|------|----------|------|------|
| 阿里云 | 0.045元/条 | 稳定、国内首选 | 需备案域名 |
| 腾讯云 | 0.04元/条 | 便宜、生态好 | 审核较严 |
| 极光 | 0.035元/条 | 便宜 | 到达率略低 |

**推荐**：阿里云短信服务，0.045元/条，1000条起购约45元。

**账号数据结构**：

```typescript
interface User {
  id: string              // UUID
  phone?: string          // 手机号（脱敏存储）
  email?: string          // 邮箱
  createdAt: Date         // 注册时间
  activationCount: number // 激活设备数
  activatedDevices: string[] // 设备指纹列表
}
```

**激活码与账号绑定**：
1. 用户注册后购买激活码
2. 输入激活码时关联当前账号
3. 激活码支持最多3台设备
4. 数据不自动同步（多设备用户需手动导出/导入）
