# 编译器 - Vercel 部署指南

## 一、项目概述

「编译器」是一个面向职场人的本地化知识管理工具，基于 React + TypeScript + Vite 构建。

## 二、快速部署（推荐）

### 方式一：GitHub 导入 Vercel（最简单）

1. **上传代码到 GitHub**
   ```bash
   cd 编译器
   # 初始化 Git 仓库（如果是新项目）
   git init
   git add .
   git commit -m "Initial commit"
   
   # 创建 GitHub 仓库并推送
   gh repo create compiler --public --push
   ```

2. **登录 Vercel 导入项目**
   - 访问 [vercel.com](https://vercel.com)
   - 点击 "New Project"
   - 选择 "Import Git Repository"
   - 选择你刚创建的 GitHub 仓库
   - Vercel 会自动检测为 Vite 项目

3. **配置环境变量（重要！）**
   - 在 Vercel 项目设置中找到 "Environment Variables"
   - 添加以下变量：
     ```
     VITE_AI_PROVIDER = douban
     VITE_AI_API_KEY = your_api_key_here
     VITE_AI_MODEL = doubao-seed-2.0-lite
     ```
   - API Key 获取方式：
     - **豆包（推荐）**：https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey
     - **DeepSeek**：https://platform.deepseek.com/api_keys

4. **部署**
   - 点击 "Deploy"
   - 等待构建完成（约 2-3 分钟）
   - 获取分配域名或绑定自定义域名

### 方式二：本地 CLI 部署

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 进入项目目录
cd 编译器

# 3. 登录 Vercel
vercel login

# 4. 构建项目
npm run build

# 5. 部署到预览环境
vercel

# 6. 部署到生产环境
vercel --prod
```

## 三、环境变量配置

### 为什么需要配置环境变量？

`.env` 文件中的 AI API Key 是敏感信息，**不能提交到代码仓库**。部署时需要在 Vercel 控制台配置。

### 必需的环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `VITE_AI_PROVIDER` | AI 服务商 | `douban` 或 `deepseek` |
| `VITE_AI_API_KEY` | API Key（必填） | `sk-xxxxxxxxxxxx` |
| `VITE_AI_MODEL` | 模型名称（可选） | `doubao-seed-2.0-lite` |

### 配置步骤

1. 登录 Vercel 控制台
2. 选择你的项目
3. 点击 "Settings" → "Environment Variables"
4. 点击 "Add New"
5. 填写变量名和值
6. 选择生效环境（Production, Preview, Development）
7. 点击 "Save"
8. **重新部署**（修改环境变量后需要）

## 四、绑定自定义域名（可选）

1. 在 Vercel 项目设置中点击 "Domains"
2. 输入你的域名（如 `app.yourdomain.com`）
3. 按提示在域名 DNS 中添加 CNAME 记录
4. 等待 DNS 生效（通常几分钟到几小时）
5. 验证并启用

## 五、验证部署

部署完成后访问你的 Vercel 域名，测试以下功能：

- [ ] 创建新笔记

## 六、常见问题

### Q: 部署后 AI 功能不工作？

**A:** 检查以下几点：
1. 环境变量是否正确配置（特别是 `VITE_AI_API_KEY`）
2. 是否重新部署过（修改环境变量后需要重新部署）
3. 查看 Vercel 函数的日志排查错误

### Q: 构建失败？

**A:** 常见原因：
1. Node.js 版本不兼容 → 确保使用 Node.js 18+
2. 依赖安装失败 → 删除 `node_modules` 和 `package-lock.json` 后重试

### Q: 如何更新部署？

**A:** 
- 如果是 GitHub 导入：推送代码后 Vercel 会自动构建部署
- 如果是 CLI 部署：运行 `vercel --prod`

### Q: 域名已备案但无法访问？

**A:** 中国大陆域名需要 ICP 备案才能绑定 Vercel。可以使用香港/海外节点，或使用 Vercel 分配的免费域名。

## 八、安全注意事项

1. **不要提交敏感信息**

2. **生产环境建议**
