### 产品简介

**你还在为这些问题头疼吗？**

- 写周报时想不起这周干了啥，翻遍聊天记录也没凑够三行

**编译器，是来解决这些问题的。**

这不是又一个「知识管理神器」。我们不想让你学会什么方法论，也不想逼你养成什么习惯。它只是一个安静的工具，帮你把脑子里的碎片整理清楚，关键时刻能找到、想用时能调出来。

### 核心功能

**情绪清洗**
有时候你只是想吐槽两句——「烦死了」「真无语」——但汇报、邮件、周报里不能这么写。编译器帮你把情绪化表达转成专业表述，不用再自己憋词。
- 内置200+情绪词→专业词映射
- 支持自定义扩展
- 一键清洗，保留原意

**一键周报**
选几篇这周的笔记，点一下，自动整合成结构化的周报。不用再翻聊天记录凑字数，你写过的东西就是你的工作证明。
- 自动按时间线排序
- 智能提取关键事项
- 支持复制/下载/发送邮件

**双向笔记**
笔记之间自动关联。写一条关于「XX项目」的笔记，它会自动关联到相关的内容。回头找的时候，能顺着线索挖出一整串相关记录。
- `[[标题]]`语法创建链接
- 自动建立反向引用
- 关系图谱可视化

**版本历史**
每次修改都有记录，随时可以回溯。不怕改坏了，反正能找回。
- 最多保留20个版本
- 支持版本对比
- 一键恢复任意版本

**离线优先**
所有数据存在本地，不上云、不依赖网络。数据完完全全属于你自己。
- 基于IndexedDB存储
- 支持导出Markdown/HTML/PDF
- 基础功能完全免费，无需激活

**智能搜索**
全文搜索+高级筛选，找东西快得多。
- 关键词高亮
- 按时间/标签/类型筛选
- 搜索性能：10000条笔记<0.5秒

**笔记模板**
内置常用模板，快速启动。
- 会议纪要、日报、周报
- 复盘、问题记录
- 支持自定义模板

### 编译真相架构

这是编译器最核心的理念，区别于传统笔记工具：

**上线区**（结论区）
- AI自动提炼的核心结论
- 当前最佳判断
- 可随时重写更新
- 类似"编译器"的执行结果

**下线区**（记录区）
- 你的原始笔记
- 只追加不修改
- 保留思考痕迹
- 类似"源代码"

**知识会过时，证据不会**
当新信息到来，结论区会更新，但记录区的原始思考永远保留。你可以随时回溯某个时间点的知识状态。

## V2EX/技术社区版

### 一个不做 AI 的知识整理工具

看到「AI」两个字你可能已经想点离开了——别急，编译器不卷大模型，不吹「第二大脑」，不卖 199 的 prompt 课。

它解决的是一个更土的问题：**你记的东西，为什么回头找不到了？**

### 核心思路

- **双向链接**：写笔记时自动建立关联，像 wiki 但没那么多手动操作
- **情绪清洗**：把「烦死了」转成「需要进一步沟通确认」，汇报时不尴尬
- **一键周报**：选笔记 → 点按钮 → 出周报，不依赖 AI 幻觉，用你自己写的原文
- **版本历史**：git 怎么用，笔记就怎么存，改坏了随时 rollback

### 编译真相架构

这不是什么花哨的概念，就是一个朴素的工程思想：

```python
class Note:
    conclusion: str  # 当前最佳判断，可更新
    records: List[str]  # 原始记录，只追加
    timestamp: int  # 记录时间

    def update_conclusion(self, new_conclusion):
        # 只更新结论，不修改记录
        self.records.append(f"Updated from {self.conclusion}")
        self.conclusion = new_conclusion
```

你想想，git 历史不就是这样的吗？你随时可以回溯某个 commit 的状态，但 commit 本身永远不会变。编译器把这套思想搬到笔记上而已。

### 技术细节

**数据格式**
- Markdown + JSON 元数据
- 可导出、可迁移
- 兼容其他 Markdown 编辑器

**存储架构**
```
indexedDB (浏览器数据库)
├── notes/           # 笔记表
│   ├── id
│   ├── title
│   ├── content      # Markdown
│   ├── metadata     # JSON (tags, links, versions)
│   └── created_at
├── links/           # 双向链接索引
│   ├── source_id
│   └── target_id
├── versions/        # 版本历史
│   ├── note_id
│   ├── content
│   └── snapshot_at
└── search_index/    # 搜索索引
    ├── keyword
    └── note_ids[]
```

**性能指标**
- 写入：<10ms
- 读取：<50ms
- 搜索：10000条笔记 <0.5s
- 版本对比：<100ms

**离线优先 ≠ 功能残缺**

本地存储不是「单机版」的遮羞布。该有的搜索、关联、回溯都有，只是数据在你自己硬盘上，不在别人服务器里。

对于程序员来说，这意味着：
- 你的笔记可以被 git 管理
- 不担心服务挂了笔记也没了
- 完全可控，数据不经过第三方
- 不用担心数据隐私问题
- 激活码离线验证，不依赖联网服务

### 为什么要做这个工具？

说实话，市面上知识管理工具已经够多了，Obsidian、Notion、Roam Research、Logseq……为什么还要造轮子？

因为它们都太"重"了。

Obsidian 需要学插件、配社区；Notion 数据在别人服务器；Roam Learning Curve 太陡峭。它们都想让你"重构你的知识体系"，但大多数人只是想把笔记记好、能找得到、用得上。

编译器不想改变你的思考方式，它只是想让你少做点体力活：
- 不用自己憋词写周报
- 不用翻聊天记录找东西
- 不用担心数据安全

就是这么简单。

**口号说「你是你唯一的算法」**，意思是：工具只负责整理和呈现，你的思考方式不会被工具绑架。
